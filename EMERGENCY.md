## 🔴 Scenario 1 — แอดโดนแบน / บัญชีถูกระงับ (Ad Rejected / Account Disabled)

### ทำทันที
1. **อย่าสร้างบัญชีใหม่มั่ว** — ยิ่งทำ Facebook ยิ่งจับ
2. เช็คว่าโดนระดับไหน:
   ```python
   r = api_get(f"{ACCOUNT_ID}", {"fields": "account_status,disable_reason"})
   # account_status: 1=ปกติ 2=ถูกระงับ / disable_reason บอกสาเหตุ
   ```
3. เข้า Ads Manager → หา Ad/Account ที่โดน → กด **Request Review / อุทธรณ์**
4. รอผล 24–48 ชม. (ปกติ)

### สาเหตุที่เจอบ่อย (ธุรกิจบ้าน/บริการ)
| สาเหตุ | แก้ |
|--------|-----|
| คำโอ้อวดเกินจริง ("ถูกที่สุด", "รับประกัน 100%") | แก้ copy ให้ neutral |
| รูปมีลิขสิทธิ์ (โหลดจากเน็ต) | ใช้รูปงานตัวเอง |
| บัญชีใหม่ใช้งบพุ่งเร็ว | ค่อยๆ เพิ่มงบ อย่า spike |
| Landing/เพจไม่ตรงกับแอด | ทำให้ตรงกัน |
| อุทธรณ์ครั้งเดียวไม่ผ่าน | อุทธรณ์ซ้ำพร้อมอธิบาย context |

## 🔴 Scenario 2 — งบหมดกลางเดือน / กลางสัปดาห์

### ทำทันที
1. เช็คว่าใช้ไปเท่าไหร่ vs แผน:
   ```python
   d = api_get(f"{ACCOUNT_ID}/insights",
       {"fields": "spend", "time_range": json.dumps({"since":"2026-07-01","until":"2026-07-05"})})
   ```
2. **อย่าเพิ่งเพิ่มงบ** — ดูก่อนว่าคุ้มมั้ย (CPR อยู่ในเป้ามั้ย)
3. ถ้า CPR ดี + อยากยิงต่อ → เสนอหัวหน้าเพิ่มงบ (ต้องยืนยัน)
4. ถ้า CPR แพง → **ปิดก่อน** ไป `3-optimize.md` แก้ แล้วค่อยเปิดใหม่

### กันล่วงหน้า
- ตั้งงบรายวันชัด (daily_budget) — Facebook ไม่เกินให้
- ดูรายสัปดาห์ทุกจันทร์ (`2-weekly.md`) จับก่อนงบบาน

## 🔴 Scenario 3 — ยอดดิ่งกะทันหัน (แชทหาย / CPR พุ่ง)

### ทำทันที
1. เช็คว่าแอดยัง **ACTIVE** จริงมั้ย (บางทีโดน pause เอง / โพสต์ถูกลบ):
   ```python
   ads = api_get(f"{ACCOUNT_ID}/ads", {"fields": "name,status,effective_status"})
   # effective_status บอกสถานะจริง เช่น DISAPPROVED, PAUSED
   ```
2. เช็ค Frequency — ถ้า > 3x = คนเบื่อรูป → เปลี่ยนรูป (`3-optimize.md` 2.3)
3. เช็คว่าเพิ่งแก้อะไรไป 3 วันก่อนมั้ย → อาจรีเซ็ต Learning → รอให้นิ่ง
4. เช็คปุ่มทักแชทยังกดได้มั้ย (ทดสอบทักเอง)

### สัญญาณเตือนล่วงหน้า (จับใน 2-weekly.md)
| สัญญาณ | ทำก่อนสาย |
|--------|----------|
| CPR เพิ่ม > 50% | เตรียมรูปใหม่ |
| Frequency > 3x | เปลี่ยนรูปใน 7 วัน |
| CPM เพิ่ม 50% | ขยายกลุ่ม |
| แชท = 0 หลายวัน | ตรวจสถานะแอด + ปุ่ม |

## 🔴 Scenario 4 — จู่ๆ ดึงข้อมูลไม่ได้ / ผู้ช่วยบอก "ไม่เจอ token" (เจอบ่อยสุด!)

> อาการ: เมื่อวานยังใช้ได้ วันนี้ error `Invalid OAuth` / `token expired` / "ไม่เจอ token"

**สาเหตุ 99% = Token หมดอายุ** (Token สั้นอายุ 1-2 ชม. · ตัวยาว 60 วัน)

### ทำทันที
1. ไม่ต้องตกใจ — ไม่ใช่บัญชีพัง แค่กุญแจหมดอายุ
2. ทำ token ใหม่ (ดู `SETUP.md` Step 2) → **แลกเป็นตัว 60 วัน** เลย
3. วางตัวใหม่ทับใน `.env` (ที่เดิม) → ใช้ได้ต่อทันที
4. บอกผู้ช่วย: **"ช่วยแลก token 60 วันให้ใหม่"** ก็ได้

> 💡 กันล่วงหน้า: ใช้ token 60 วันเสมอ · จดวันหมดอายุไว้ · ครบ 60 วันทำใหม่ก่อนตาย

## 🔴 Scenario 5 — แอดภาษาไทยกลายเป็นต่างดาว (copy เพี้ยน)

> อาการ: ให้ Claude เขียนแอดไทย แต่ในแอด/โพสต์จริงเป็น `à¸šà¹‰à¸²à¸™` หรือ `บ` — อ่านไม่ออก
> สาเหตุ = **encoding เพี้ยนตอนส่งข้อความ** (พิมพ์ไทยยาว inline ใน shell locale ไม่ใช่ UTF-8 / อ่านไฟล์ผิด encoding)

### ทำทันที
1. **อย่าเปิดแอด** ถ้ายังเพี้ยน — คนเห็นแล้วเสียภาพ + เสียงบ
2. เขียน copy ลงไฟล์ UTF-8 ใหม่ แล้วอ่านกลับด้วย `encoding="utf-8"` (ดู `1-setup.md` Step 6.5 ชั้น 1)
3. รันสคริปต์ด้วย `PYTHONUTF8=1 python3 script.py`
4. **อ่าน message กลับมาเช็ค** ก่อนเปิด:
   ```python
   check = api_get(f"{POST_ID}", {"fields": "message"}, token=PAGE_TOKEN)
   print(check.get("message"))   # อ่านไทยออก = OK
   ```
5. ถ้าโพสต์เพี้ยนไปแล้ว → ลบโพสต์/creative นั้น สร้างใหม่จากไฟล์ UTF-8

### กันล่วงหน้า
- เขียน copy ลงไฟล์เสมอ — ห้ามพิมพ์ไทยยาวๆ ใส่ตรงใน command/`-c`
- อ่าน message กลับมาดูทุกครั้งก่อนเปลี่ยนเป็น ACTIVE
