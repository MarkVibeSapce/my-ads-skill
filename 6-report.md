# Module 6 — รายงานส่งหัวหน้า (Weekly / Monthly · PDF + PPTX + HTML)
> ออกรายงานผลแอดพร้อมส่ง 3 รูปแบบ ดีไซน์เดียวกัน · light mode · ฟอนต์ Sarabun
> คู่กับสไลด์ P5 (Weekly) + P6 (Monthly)

> ⚙️ ใช้ `report/gen_report.py` — ดึงตัวเลขจาก API เอง + อ่าน token/ACCOUNT_ID จาก `.env`

---

## 💡 3 รูปแบบ เอาไปทำอะไร

| ไฟล์ | เหมาะกับ |
|------|---------|
| **PDF** | ส่ง LINE / อีเมล หัวหน้า · แนบรายงาน · พิมพ์ |
| **PPTX** | เอาขึ้นจอประชุม · หัวหน้าอยากแก้ต่อเอง |
| **HTML** | เปิดดูเร็วในเบราว์เซอร์ · กดพิมพ์เป็น PDF เองก็ได้ |

> ทั้ง weekly + monthly ดีไซน์เดียวกัน — weekly กระชับกว่า (ไม่มี funnel/inbox)

---

## 🧠 หลักคิด: แยก "ตัวเลข" ออกจาก "บทวิเคราะห์"

```
ตัวเลข   → ดึงจาก Meta API (spend/แชท/CPR/reach/รายกลุ่ม)  ← script ทำเอง ห้ามเดา
วิเคราะห์ → สรุปผู้บริหาร / สิ่งที่ดี / ต้องปรับ / แผนต่อไป   ← Claude เขียนจากตัวเลข
```
สองอย่างมาเจอกันใน `report/report_data.json` แล้ว render ออก 3 ไฟล์

---

## ⚡ ทางลัด — ดูตัวอย่างทันที (ไม่ต้องต่อ API)

```bash
cd report
python3 gen_report.py --period monthly --mock --out all   # ออก 3 ไฟล์ตัวอย่าง
python3 gen_report.py --period weekly  --mock --out all
```
> ได้ `รายงานแอด-เดือน.{html,pdf,pptx}` เลขสมมติ "บ้านในฝัน" — ดูหน้าตาก่อนใช้จริง

---

## 🚀 ใช้จริง — 3 ขั้น

### ขั้น 1 — ดึงตัวเลขจริง
```bash
cd report
python3 gen_report.py --period weekly --fetch      # หรือ --period monthly
```
→ เขียน `report_data.json` (ตัวเลขครบ · ช่องวิเคราะห์ว่างไว้)
> ช่วงเวลา default: weekly = 7 วันล่าสุด · monthly = เดือนที่แล้วเต็มเดือน
> กำหนดเองได้: `--since 2026-07-01 --until 2026-07-07`

### ขั้น 2 — Claude เติมบทวิเคราะห์
เปิด `report_data.json` เติมช่องเหล่านี้ (อ่านจากตัวเลข + inbox):
| ช่อง | ใส่อะไร |
|------|--------|
| `exec_summary` | สรุปผู้บริหาร 2–3 ประโยค (ใช้ `**ตัวหนา**` เน้นได้) |
| `goods` | สิ่งที่ทำได้ดี (list) |
| `improves` | สิ่งที่ต้องปรับ (list) |
| `next_plan` | แผนช่วงหน้า 1–2 อย่างชัดๆ |
| `funnel` | (monthly) `[["แชททั้งหมด",112],["เลดจริง",68],...]` |
| `questions` | (monthly) `[["ราคา / งบ",34],...]` — จาก inbox (Module 5) |
| `leads`,`lead_rate` | เลดจริง + % (จาก inbox) |

> ตัวเลข spend/แชท/CPR/persona **อย่าแก้** — มาจาก API จริงแล้ว
> วิเคราะห์ตามจริง เล่าจริง > เลขสวย (ถ้าผลแย่ก็บอกตรง + วิธีแก้)

### ขั้น 3 — ออกไฟล์
```bash
python3 gen_report.py --period weekly --render --out all   # html+pdf+pptx
# หรือแยก: --out pdf | --out pptx | --out html
```

## 🎨 มาตรฐานดีไซน์ (แบบเอเจนซี่มืออาชีพ)
รายงานใช้โครงเดียวกับรายงานเอเจนซี่จริง:
- **Header ribbon เข้ม** + เส้น accent สีแบรนด์ (ทุกสไลด์/หัวรายงาน)
- **KPI card** มีแถบ accent + ค่าตัวใหญ่ · **ตารางหัวเข้ม** · body light mode
- **สไลด์:** หน้าปก → สรุปผู้บริหาร → คุณภาพแชท → รายกลุ่ม → สรุป&แผน
- วัดผล **ตรงธุรกิจแชท** (CPR/เลดจริง/depth/persona) ไม่ใช่ตัวเลขทั่วไป

## ⚙️ ตั้ง CI แบรนด์ใน `.env` (รายงานออกสีแบรนด์คุณเอง)
```
ACCOUNT_ID=act_xxxxxxxx      # จาก BRAND_PROFILE ข้อ 2
BRAND_NAME=บ้านในฝัน ดีไซน์
BRAND_TAGLINE=รับออกแบบและสร้างบ้าน
BRAND_ACCENT=2E5BE6          # สีหลักแบรนด์ (hex ไม่ต้องมี #) — เน้นค่า/เส้น/การ์ด
BRAND_DARK=16202E            # สี ribbon หัวสไลด์ (เข้ม)
CPR_TARGET=150               # เป้า CPR — ใช้ตัดสถานะ ✅/🔴
REPORT_EN_FONT=Sarabun       # ฟอนต์ตัวเลข (default ปลอดภัยทุกเครื่อง)
```
> ไม่ใส่ก็ได้ — มี default ให้ · เปลี่ยน `BRAND_ACCENT` = รายงานเปลี่ยนโทนสีทั้งชุดทันที
> (script คำนวณเฉดเข้ม/อ่อนจากสีแบรนด์ให้เอง — ใส่สีเดียวพอ)
